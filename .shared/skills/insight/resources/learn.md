# learn

Find 3-7 high-leverage skills where focused investment produces disproportionate returns.

## Core Definition

Identify "3-7 things where investing 50-100 hours of deep learning or practice would produce step-function changes in capability, output, or trajectory."

**ROI:** 10x-100x, not incremental

**Verification Test:** "A true leverage point, when developed, unlocks progress in 3+ areas simultaneously."

## Five-Phase Framework

### Phase 1: Constraint Mapping

Diagnose through:
- Structural scans of vault folders and notes
- Context file analysis (ruthless priorities, project MOCs, scope overviews)
- 30-day daily note reviews
- Behavioral archaeology (grep for decision patterns, repeated frictions)
- Project audits (active projects, stalled work, deferred decisions)

**Identify:**
- Recurring frictions
- Capability gaps
- Dependency points
- Deferred decisions

### Phase 2: Leverage Detection

**Six methods (order of reliability):**

1. **Bottleneck convergence** - Skills blocking 3+ domains
2. **Adjacent possible** - 80% developed capabilities needing a final push
3. **Multiplier models** - Frameworks that reshape decisions across areas
4. **Removal leverage** - Skills that automate or eliminate recurring bandwidth drains
5. **Relationship leverage** - Network amplification capabilities
6. **Identity leverage** - Self-limiting beliefs masquerading as capability gaps

### Phase 3: Beyond the Vault

Identify solutions in "domains outside current awareness":
- Blind spot analysis — what topics are conspicuously absent given stated priorities?
- Outsider perspectives — what would someone from a different field suggest?
- Imported frameworks from fields that already solved similar constraints

### Phase 4: Verification

**Six filters — each candidate must pass:**
- **3+ domain test:** Does developing this unlock progress in 3+ areas simultaneously?
- **Evidence test:** Does vault evidence prove this is a real constraint, not assumed?
- **Feasibility test:** Is 50-100 hours a realistic development path for this skill?
- **Counterfactual test:** What specifically stays broken if this skill isn't developed?
- **Already-tried test:** Has this been attempted before? If so, what was different then?
- **Compound test:** Does this skill make other skills more effective over time?

### Phase 5: Output

Leverage map with 3-7 ranked findings. For each:

**Skill:** [Name — specific, not vague]

**Source:** [Which detection method surfaced this]

**Constraint Addressed:** [What problem this solves, with vault evidence]

**Domains Unlocked:** [3+ areas that become unblocked]

**Counterfactual Impact:** [What stays broken without this]

**Development Path:** [Concrete 50-100 hour learning approach]

**Confidence:** [High / Medium / Low — based on evidence quality]

---

## What This Mode Is NOT

- Broad idea generation
- Surfacing implicit ideas (that's `ideate`)
- Finding domain bridges (that's `nexus connect`)
- Measuring intention-behavior gaps (that's `align`)
- Identifying revenue opportunities

## Success Criteria

- **Specificity:** Not "communication" — "structured async writing for distributed decision-making"
- **Constraint-evidence:** Vault proves the problem exists
- **Multi-domain impact:** Unlocks 3+ areas, not just one
- **Concrete development paths:** Clear 50-100 hour plan, not vague "explore this"

## Anti-Patterns

- Self-help generics
- Interesting-but-irrelevant domains
- Polishing existing strengths
- Unrealistic time requirements
- Single-domain skills
- Motivation-based reasoning
- Vague guru recommendations
- Meta-skill spirals ("learn how to learn")
- Vault-ceiling thinking (only looking at what's already in the vault)

## Execution

You should:
1. Map constraints using vault file system — read context files, diary entries, project notes directly
2. Apply all 6 leverage detection methods
3. Look beyond vault for blind spots
4. Filter candidates through all 6 verification tests
5. Output ranked leverage map with 3-7 skills

**Remember:** The strongest leverage points "feel slightly uncomfortable" and often surprise through unexpected domain imports. If the output feels obvious, dig deeper.
