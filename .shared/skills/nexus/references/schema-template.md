# NEXUS.md Schema Template

This template is used by the `init` mode to generate a project-specific `NEXUS.md` file. Variables in `{curly braces}` are replaced with user-provided or default values during init.

The generated `NEXUS.md` is placed at the Nexus root and serves as the schema file that the LLM reads before every operation.

---

## Template

```markdown
---
domain: {domain}
purpose: {purpose}
created: {ISO date}
nexus-root: {nexus root path}
ingest-mode: {interactive|batch}
crossref-style: {wikilinks|markdown}
---

# {Domain} — Nexus

## Overview

{One-paragraph description of what this knowledge base covers and its purpose.}

## Directory Structure

| Directory | Purpose |
|-----------|---------|
| `raw/` | Immutable source documents. The LLM reads but never modifies these. |
| `raw/assets/` | Images and attachments from sources. |
| `pages/` | LLM-generated and maintained pages. |
| `pages/{sources-subdir}/` | Source summary pages. One per ingested source. |
| `pages/{entities-subdir}/` | Entity pages (people, organizations, tools, etc.). |
| `pages/{concepts-subdir}/` | Concept and topic pages. |
| `pages/{synthesis-subdir}/` | Cross-cutting analysis, comparisons, query results, discoveries. |
{additional-directories}

## Page Types

| Type | Location | Genesis | Description |
|------|----------|---------|-------------|
| `source` | `pages/{sources-subdir}/` | `source` | Summary of an ingested source with key claims and entity links |
| `entity` | `pages/{entities-subdir}/` | `derived` | A person, organization, tool, place, or other named thing |
| `concept` | `pages/{concepts-subdir}/` | `derived` | A topic, idea, method, or abstract concept |
| `synthesis` | `pages/{synthesis-subdir}/` | `synthesis` | Cross-source analysis, comparisons, or filed query answers |
| `emergent` | `pages/{synthesis-subdir}/` | `emergent` | Ideas surfaced by discover mode from pattern analysis |
{additional-page-types}

## Genesis Values

| Value | Set by | Meaning |
|-------|--------|---------|
| `source` | ingest | Direct extraction/summary from a raw source |
| `derived` | ingest | Entity/concept page created during ingestion |
| `synthesis` | query | Filed answer to a user query |
| `emergent` | discover | Surfaced by pattern analysis across existing pages |
| `manual` | user | Created or edited directly by the user |

## Conventions

### Cross-References
- Style: {wikilinks `[[Page Name]]` | markdown links `[text](path)`}
- Every page links to at least one other page.
- Source pages link to entities and concepts they mention.
- Entity and concept pages link back to sources that reference them.
- Bidirectional: if A links to B, B links back to A.
- Emergent pages link to the pages whose patterns they were derived from.

### Frontmatter
All pages include YAML frontmatter with at minimum:
- `type`: page type from the table above
- `genesis`: how this page was created (see Genesis Values)
- `created`: ISO date of creation
- `updated`: ISO date of last modification
- `sources`: list of source pages that inform this page
- `tags`: domain-relevant tags
{additional-frontmatter-fields}

### Naming
- Page names: {naming convention, e.g., title case, kebab-case, etc.}
- File names: {file naming convention}
{additional-naming-rules}

## Ingest Workflow

- Mode: {interactive|batch}
- Auto-create entity/concept pages: {yes|no}
- Discussion before integration: {yes|no|only for contradictions}
{additional-ingest-rules}

## Domain-Specific Rules

{Any domain-specific conventions, terminology standards, or special handling rules.

## Notes
```
